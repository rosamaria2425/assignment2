8. Inheritance:
Inheritance allows a class to inherit properties and methods from another class so code can be reused.

Polymorphism:
Polymorphism means a method can behave differently depending on the object that is calling it.

Encapsulation:
Encapsulation means keeping data inside a class private and accessing it through public methods.

9. The main difference between the ‘Dictator and Lieutenants’ workflow and the ‘Integration manager’ workflow is related to code change integration. In ‘Dictator and Lieutenants,’ the project lead is considered the dictator who decides what is the final code. The lieutenants are responsible for handling other developers' contributions before passing them on to the dictator. In ‘Integration manager,’ developers fork the code and send pull requests to the integration manager.

10. The team of 100 developers will benefit from the “Dictator and Lieutenants” workflow because it will help them organize their contributions through different people instead of everyone directly contributing to one repository. The lieutenants will review and oversee changes from different teams of developers before they are submitted to the main project leader. This will reduce confusion and conflicts, and keep everything organized compared to a centralized approach.
